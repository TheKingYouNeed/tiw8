Guide des fonctionnalités - Application Q&R

Ce guide vous explique comment tester les différentes fonctionnalités de l'application de questions-réponses développée pour le TP2 du module TIW8.

Note importante : Assurez-vous que le serveur et le client sont en cours d'exécution avant de tester les fonctionnalités.

Architecture SPA (Single-Page Application)

1. Application React avec TypeScript

  1. Naviguez entre les différentes vues sans rechargement de page
  2. Observez la mise à jour fluide de l'URL dans la barre d'adresse

2. Composants UI modulaires

  1. Examinez la structure des composants dans le dossier client/src/components/
  2. Notez comment chaque composant (toolbar, event panel, question cards) est réutilisable

3. Structure statique avec données dynamiques

  1. Créez un nouvel événement et observez comment l'interface se met à jour sans rechargement

Gestion des événements

1. Affichage de la liste des événements

  1. Accédez à http://localhost:5173/events pour voir la liste des événements

2. Sélection d'un événement

  1. Cliquez sur un événement dans la liste pour voir ses questions
  2. Notez comment l'URL change pour refléter l'événement sélectionné (/event/:eventId)

3. Ajout de nouveaux événements

  1. Cliquez sur "Créer un nouvel événement"
  2. Remplissez le formulaire et soumettez-le
  3. Vérifiez que le nouvel événement apparaît dans la liste

Gestion des questions

1. Affichage des questions

  1. Sélectionnez un événement pour voir ses questions
  2. Observez les propriétés de chaque question (couleur, contenu, auteur, etc.)

2. Création de questions

  1. Dans la vue d'un événement, cliquez sur "Ajouter une nouvelle question"
  2. Remplissez le formulaire et soumettez-le
  3. Vérifiez que la nouvelle question apparaît dans la liste

3. Vote sur les questions

  1. Cliquez sur les boutons de vote (↑/↓) sur une question
  2. Observez le changement du nombre de votes

Flux de données inversé

1. Notification des composants parents

  1. Sélectionnez un événement et observez comment tous les composants enfants sont mis à jour
  2. Votez sur une question et observez la mise à jour de l'état global

2. État global

  1. Ouvrez les outils de développement Redux (extension navigateur)
  2. Observez l'état currentEvent et currentQuestion se mettre à jour lors de vos interactions

Routage basé sur les URL

1. Mode administrateur

  1. Accédez à http://localhost:5173/admin/event/:eventId
  2. Vérifiez que vous avez accès à toutes les fonctionnalités d'édition

2. Mode participant

  1. Accédez à http://localhost:5173/participant/event/:eventId
  2. Vérifiez que certaines fonctionnalités d'édition sont désactivées

3. Accès direct à une question

  1. Accédez à http://localhost:5173/event/:eventId/question/:questionId
  2. Vérifiez que la question spécifique est affichée

4. Navigation entre questions

  1. Utilisez les flèches "‹" et "›" pour naviguer entre les questions
  2. Observez le changement d'URL et la mise à jour de l'interface

État global avec Redux Toolkit

1. Store Redux

  1. Ouvrez les outils de développement Redux
  2. Examinez la structure du store et les différents états

2. Actions typées

  1. Effectuez différentes actions (sélection d'événement, vote, etc.)
  2. Observez les actions dispatched dans les outils Redux

3. Hooks React-Redux

  1. Examinez le code source pour voir l'utilisation de useSelector et useDispatch

Middleware

1. Logger

  1. Ouvrez la console du navigateur
  2. Effectuez différentes actions et observez les logs détaillés (état précédent, action, état suivant)

2. Propagateur WebSocket

  1. Ouvrez l'application dans deux onglets ou navigateurs différents
  2. Effectuez une action dans un onglet et observez la synchronisation dans l'autre

Synchronisation multi-dispositifs en temps réel

1. Diffusion des actions

  1. Ouvrez l'application sur plusieurs appareils connectés au même réseau
  2. Effectuez des modifications sur un appareil et observez la mise à jour en temps réel sur les autres

2. Mise à jour instantanée

  1. Créez une question sur un appareil
  2. Vérifiez qu'elle apparaît immédiatement sur les autres appareils

Design adaptatif

1. Mise en page adaptative

  1. Redimensionnez la fenêtre du navigateur ou utilisez les outils de développement pour simuler différents appareils
  2. Observez comment l'interface s'adapte aux différentes tailles d'écran

2. Mode plein écran

  1. Cliquez sur le bouton de plein écran dans l'interface
  2. Vérifiez que l'application s'affiche correctement en plein écran

Canvas et reconnaissance de gestes

1. Dessin à main levée

  1. Dans la vue d'un événement, utilisez le canvas pour dessiner
  2. Essayez de dessiner les gestes "‹" et "›"

2. Reconnaissance de gestes

  1. Dessinez le geste "‹" pour naviguer vers la question précédente
  2. Dessinez le geste "›" pour naviguer vers la question suivante
  3. Vérifiez que la navigation fonctionne correctement après la reconnaissance du geste
Astuce : Pour de meilleurs résultats, dessinez les gestes d'un seul trait continu et essayez de les faire ressembler autant que possible aux symboles "‹" et "›".
